# 反馈系统（Feedback System）

来自[[26_06_06 考试结束的反思与成长仪表盘]]的核心发现：

> **Ethan并非没有进步，而是长期只用体重作为唯一评价标准，导致大量真实成长被忽略，从而逐渐失去了对自己进步的感知能力。**

## 问题的根源

> 如果没有体重秤，我其实很难判断是否进步。

→ 因此产生"我是不是根本没有进步？"的怀疑。

**痛苦不来自体重，而来自缺少能看见自己成长的反馈系统。**

## Ethan成长仪表盘（Growth Dashboard）

多维度记录，不再只靠体重一个指标：

**身体：** 体重趋势 / 腰围 / 月度照片

**力量：** 卧推 / 下拉 / 腿部数据

**有氧：** 持续运动时间

**学习：** 最长专注时间

**心理：** 情绪评分 / 精力评分

## 2026年咨询中的深化

- [[26_05_31 痛苦作为安全感]]：体重从103kg短期涨到104.5kg时，情绪会迅速滑向“我是不是失败了”。这说明体重数字一旦成为唯一反馈，就会把水分波动误读成人生评价。
- [[26_06_11 沉默期与自己给自己供电]]：在上海图书馆读到关于[[沉默期]]和无反馈坚持的文章后，意识到反馈系统很重要，但也要能承受反馈暂时不出现的阶段。反馈是结果给的礼物，不是行动的前提。
- [[26_06_18 同居后的幸福与第一块GitHub绿砖]]：第一次点亮 GitHub contribution 的小绿框，说明反馈系统不只可以记录体重和训练，也可以记录“我今天留下了痕迹”。
- [[26_06_19 把模糊的钱找回来]]：反馈系统被推进到投资和记账场景。[[账本系统]]让每一笔投入、买入、卖出、手续费和历史调整项都能被看见，避免钱变成模糊压力。
- [[26_06_20 厨房里的幸福与一次完整的GitHub分支流程]]：GitHub 的 `commit`、`push`、`branch`、PR 和 `merge` 流程，让技术学习也变成可追踪的反馈系统。每一次分支和提交都在说明：项目不是停在想法里，而是在真实推进。
- [[26_06_20 厨房里的幸福与一次完整的GitHub分支流程]]：GitHub contribution 小绿块进一步说明，反馈系统不一定要很复杂才有效。一个每天一个格子的可视化记录，就足以让人因为“我今天留下了痕迹”而开心，并愿意继续行动。
- [[26_06_23 在难处没有退缩，也想给她未来]]：反馈系统进入学习方法本身。面对 TypeScript、Vitest 和 Golden Test 的不理解，不是停在“我不会”，而是让 AI 以程序员和面试官视角出题，再用自己的语言回答，用题目检测理解缺口。
- [[26_06_29 跳票一周后，我还是不会停下来]]：反馈系统进入项目延期后的复盘。账本变成黑盒、开发计划从 12 周推迟到 13 周时，真正有用的反馈不是“我失败了”，而是识别卡点、承认复杂度、把黑盒拆成可以继续学习和验证的小块。
