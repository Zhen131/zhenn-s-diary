# 厨房里的幸福与一次完整的GitHub分支流程

## 睡醒之后一直在开发

今天我睡醒之后，基本上就一直在开发我的[[账本系统]]项目。

从醒来到现在，大概已经开发了四五个小时。中间我一边写代码，一边看比赛，一边继续把这个项目往前推。这个账本项目现在越来越不像一个“我随便做做看”的练习，而是越来越像我生活里真的需要的东西。

我今天也练习了一次比较完整的 GitHub 分支流程：先从 `main` 开新分支，在分支上提交 Week2 源码目录骨架，再 `push` 到 GitHub，创建 PR，请求把分支合并回 `main`。

我理解了：

- `commit` 是本地存档；
- `push` 是上传到远端；
- `branch` 是独立草稿线；
- `PR` 是合并申请；
- `merge` 是把确认过的改动并回主线。

自己的项目可以直接开分支；别人的开源项目通常要先 `fork` 到自己账号，再从自己的 `fork` 向原仓库提交 PR。

这件事让我感觉到一种新的[[反馈系统]]正在建立起来。

以前我看 GitHub 的时候，它对我来说像是一个很远的程序员世界：分支、提交、PR、合并，好像都是别人每天在做的事情。可是今天我自己完整走了一遍以后，它突然变得很具体。

代码不是凭空“变好”的。它是先在一条分支上被尝试、被保存、被上传、被审查，然后再合并回主线。

这个流程本身也像一种[[主体性]]：我不是把项目弄得乱七八糟以后再补救，而是在给自己的工作留下清晰的痕迹和边界。

## 又点亮了一块 GitHub 绿块

今天还有一个很小、但让我很开心的事情：我又点亮了 GitHub 上的一个小绿块。

我突然注意到，GitHub 的 contribution 图是一天天算的。比如 17 号是一个格子，18 号是一个格子，19 号是一个格子，20 号又是一个格子。

如果这一天什么都没做，这个格子就是空的。

因为我 19 号没有做任何操作，所以 19 号的格子是空的；18 号和 20 号有绿色标志。

说实话，这个东西好像没什么特别实际的用处。它不能证明我代码写得多牛，也不能证明我真的多专业。甚至我现在虽然已经开始大量用 GitHub，但我依然感觉自己非常不专业。

可是我就是因为这个小绿块高兴得很。

这件事反而给了我一个很重要的产品启发：专业工具也需要趣味化和生活化。

GitHub 吸引我的，不只是它有多专业，而是它把“我今天做了什么”变成了一个非常直观、非常有反馈感的小东西。每天一个格子，有做事就亮起来，没做事就空着。它很简单，但它会让人想回来看看，也会让人想继续留下一点痕迹。

这正好印证了我在[[账本系统]]需求总纲里写的第八模块：趣味化和生活化模块。

我的账本项目以后也不应该只是冷冰冰地记录交易、持仓、成本、现金和手续费。它也应该有一些能让人愿意回来看的东西，比如奖章系统、连续记录、阶段成就、生活化的提示，甚至是类似小绿块一样的可视化痕迹。

这些东西看起来不“专业”，但它们真的有用。

它们不一定直接提升功能正确性，却能提升留存，提升情绪反馈，提升一个人继续使用系统的动力。

我今天的开心本身，就是证据。

如果 GitHub 这样专业的平台都需要这种小小的趣味化设计，那我的账本系统当然也可以有。它不是不专业，而是让一个专业系统更容易进入人的生活。

## 四次加时的比赛

今天还看了一场特别精彩的 CS 比赛。

这场比赛是 2020 年科隆 Major 里 Team Spirit 绿龙战队对阵法国 G2 战队。G2 里面有 Hunter 这样的明星选手，也有那个我老是念成像“螳螂”的 Matisse，还有那个太阳神狙击手。

比赛真的很精彩。

Matisse 发挥很强，有一种力挽狂澜的感觉。

最夸张的是，加时加了四次才分出胜负。那种比分一直咬住、谁都不能完全松一口气的感觉，很让人上头。

中间还出现了机器坏掉导致技术暂停的情况。G2 本来领先了三四分，手感正热，结果机器坏了以后被迫暂停。等技术暂停结束之后，手感好像突然凉了下来，然后就被一点点追分、反超。

这种剧情真的很残酷。

竞技比赛里有时候就是这样：你不是只和对手打，也在和自己的手感、节奏、设备、暂停之后的身体状态打。明明前面已经打出了优势，但一个技术暂停就可能把那种流动感切断。

最后虽然 G2 输了，但我还是觉得这是一场非常精彩的比赛。

现在我还在看小蜜蜂打 Falcons。Falcons 这个队名我还没想好中文到底怎么翻译，就先这么叫。里面有 Niko，还有 carrigan 这种我很熟悉的名字。今天这种一边开发、一边看比赛的状态，其实挺舒服的。

## 厨房里的幸福

今天最重要的感受，还是幸福。

我现在是在厨房里面写代码、看比赛、做自己的事情。

为什么在厨房里面？因为她在睡觉，我不想打扰她。所以我把厨房窗户关得严严实实的，就在这里继续开发。

这画面其实很普通：她在房间里睡觉，我在厨房里干活。

但我就是觉得很幸福。

她今天给我做了饭，还给我热饭吃。她照顾我，摸我的头发，把我当成两个小孩子一样对待。我会觉得特别感动。

那一瞬间我好像有一种“变成了男人”的感觉。

不是那种要变得强硬、不能撒娇、不能软弱的男人，而是好像突然更真实地感觉到：我有一个很爱我的人，我也在努力做自己的事情，我在慢慢建设我们的生活。

我能够有她陪着，有她在我身边，就觉得非常幸福。

这种幸福不是特别戏剧化的东西。不是一定要发生什么大事，不是一定要去哪里玩，不是一定要买什么东西。它就是她在睡觉，我在厨房里开发；她给我热饭吃，照顾我，摸我的头发；我知道她对我特别好，我也知道自己很幸运能遇到她。

我们在一起三年了。

我今天真的觉得，人生里能遇到这样的[[../2023日记/爱]]和[[../2023日记/亲密关系]]，是一件不可多得的事情。

我甚至会想，有这样的妻子，是很幸运很幸运的事。

## 一种意义感

今天让我最感动的地方，是我好像找到了一种意义感。

我不是单纯在敲代码，也不是单纯在看比赛，更不是单纯在被她照顾。

这些东西叠在一起，突然组成了一种生活：

- 我在开发一个对自己真实有用的账本项目；
- 我在学习 GitHub 的真实工作流；
- 我在看一场精彩到四次加时的比赛；
- 我吃着女朋友给我做的饭；
- 我知道她就在我身边睡觉；
- 我为了不打扰她，安安静静地在厨房里继续做自己的事情。

这就是今天最珍贵的地方。

我好像同时拥有了两种感觉：一边是被爱着、被照顾着；另一边是自己正在一点点把未来搭起来。

这让我觉得很幸运。

也让我觉得很幸福。
