# 📖 新日记入库指南
> 写给负责维护这个 Obsidian 知识图谱的 AI 助手。
> 你是这个资料库的图书管理员——负责新日记的**入库**、**贴标签**，以及对老日记的**更新标签**，让整张知识图谱始终保持连通。

## 一、资料库结构

```
日记/
├── 00 工具与指南/             ← 工具说明、入库指南、总目录书签
│   ├── AGENTS.md
│   ├── 📖 新日记入库指南.md   ← 你现在读的这个文件
│   └── 📚 日记总目录.md       ← 集中维护的目录书签
├── 2023日记/                  ← 日记条目 + 概念节点
├── 2024日记/                  ← 日记条目 + 概念节点 + 成长轨迹
├── 2025日记/                  ← 日记条目 + 概念节点 + 成长轨迹
└── 2026日记/                  ← 持续更新（日记条目 + 概念节点 + 成长轨迹）
```

每个年份文件夹主要包含三类文件：

| 类型 | 命名规则 | 示例 |
|------|----------|------|
| **日记条目** | `YY_MM_DD 标题.md` | `26_06_06 考试结束的反思.md` |
| **概念节点** | `概念名称.md` | `全能自恋.md`、`反馈系统.md` |
| **成长轨迹** | `成长轨迹XXXX→XXXX.md` | `成长轨迹2023→2024→2025→2026.md` |

年度索引已经合并为 `00 工具与指南/📚 日记总目录.md`。不要再为每一年单独创建 `📋 20XX日记索引.md`，避免工具型目录在图谱里压过真正重要的概念节点。

## 二、日记条目文件格式

每篇日记都必须有以下结构：

```markdown
---
tags: [日记/20XX, 关键词1, 关键词2]
date: 20XX-MM-DD
---

# 标题

正文内容，把重要词汇用 [[wikilink]] 格式连接……

## 关联
- [[同文件夹内的概念节点]]
- [[../20XX日记/跨年概念节点]]
```

**规则：**
- `tags` 中第一个始终是 `日记/年份`（如 `日记/2026`）
- 正文中的关键概念**内嵌** `[[wikilink]]`，不只是写在末尾
- `## 关联` 部分列出最相关的节点，跨年用相对路径 `[[../2023日记/阿德勒]]`

---

## 三、概念节点文件格式

当一个概念在**多篇日记**中反复出现，就应该为它建立独立的概念节点：

```markdown
---
tags: [概念/分类, 关键词]
---

# 概念名称

概念定义和核心内涵……

## 核心洞见（引用日记原文）

> 引用某篇日记中对这个概念最精彩的阐述

## 出现于以下日记

- [[日记文件1]]
- [[日记文件2]]

## 关联
- [[相关概念节点]]
- [[../20XX日记/跨年概念]]
```

## 四、入库流程（处理新日记的步骤）

### 第一步：读懂日记

仔细阅读用户输入的新日记，提取：
- **日期**（确定放入哪个年份文件夹）
- **核心主题**（2-4个关键词，用于 `tags`）
- **出现的概念**（哪些词需要 `[[wikilink]]`）
- **情绪/事件/洞见**（决定标题和内容结构）

### 第二步：创建日记条目文件

按格式创建 `YY_MM_DD 标题.md`，保存到对应年份文件夹。

内嵌 `[[wikilink]]`——优先连接**已存在的**概念节点（见下方概念库总览）。

### 第三步：判断是否需要新建概念节点

**需要新建概念节点的情况：**
- 这个概念是**全新的**，之前从未出现过
- 这个概念未来很可能在多篇日记中反复出现
- 这个概念是某段成长的核心转折点

**不需要新建的情况：**
- 只是偶发提及
- 已有相近的概念节点可以覆盖

### 第四步：更新相关老文件（重要！）

这一步是让新老知识图谱保持连通的关键：

**4a. 更新相关概念节点**

如果新日记深化或印证了某个老概念，在该概念节点文件末尾追加：

```markdown
## 20XX年日记中的引用/深化

- [[../20XX日记/新日记文件名]]（一句话说明新日记的贡献）
```

**4b. 更新总目录书签**

在 `00 工具与指南/📚 日记总目录.md` 对应年份/月份的表格里，添加新日记的条目。

注意：总目录只是工具型书签，负责帮助人快速找到日记；不要把它当作成长主线或概念节点来维护。真正有思想含量的连接，应优先写进概念节点和成长轨迹文件。

**4c. 按需更新成长轨迹文件**

如果新日记包含显著的成长里程碑，在最新的 `成长轨迹XXXX→XXXX.md` 中更新相关部分。

## 六、跨年链接规范

在 Obsidian 中，跨文件夹的链接必须使用**相对路径**：

```markdown
[[../2023日记/阿德勒]]          ← 从2026日记引用2023概念
[[../2024日记/全能自恋]]        ← 从2026日记引用2024概念
[[../2025日记/向内求]]          ← 从2026日记引用2025概念
```

同文件夹内的链接直接写名字：
```markdown
[[反馈系统]]                    ← 同在2026日记/文件夹内
```

从 `00 工具与指南/📚 日记总目录.md` 指向年份文件夹时，也使用相对路径：

```markdown
[[../2026日记/26_06_18 同居后的幸福与第一块GitHub绿砖]]
[[../2025日记/财商]]
```

## 七、索引降权规则

这个资料库的核心不是“年份”，而是反复出现、不断深化的概念。年度索引已经合并成一个总目录书签，目的是减少图谱中的行政节点。

**维护原则：**
- 总目录只负责收纳入口，不负责解释意义。
- 新日记的主要连接应写入日记正文、`## 关联`、相关概念节点和成长轨迹。
- 如果一个概念反复出现，应该更新或新建概念节点，而不是依赖总目录来建立连接。
- 使用 Obsidian 图谱时，可以过滤 `tag:#工具/目录` 或路径 `00 工具与指南`，让目录书签从图谱中心退场。
